# 🦾 Idempotency  
> «اگه ده بار بزنی دکمه، انگار فقط یه بار زدیش!»

---

## 🎯 Idempotency چیه؟

یه مفهوم نجات‌بخش که می‌گه:

> «اگه یه درخواست API چند بار بیاد، نتیجه نهایی باید همون باشه که فقط یک بار اومده بود.»
> «سرور: ببین، می‌دونم ده بار زدی روی Submit، ولی من فقط یه بار کار می‌کنم.  
>  آروم باش.»

---

## 😱 مشکل از کجا شروع می‌شه؟

- اینترنت کند  
- کاربر هی دکمه پرداخت رو دوباره می‌زنه  
- مرورگر می‌زنه Refresh  
- موبایل می‌ره آفلاین/آنلاین  
- درخواست Retry سمت کلاینت یا Gateway

نتیجه بدون Idempotency:
- ده بار پول کم می‌شه  
- ده تا سفارش ثبت می‌شه  
- دیتابیس می‌گه: *حالم خوب نیست، کمک.*

---

## 🧬 چه درخواست‌هایی باید Idempotent باشن؟

- پرداخت‌ها (ساخت تراکنش)  
- رزرو صندلی  
- ایجاد سفارش  
- هر کاری که **پول، پیامد، یا مسئولیت قانونی** داره  

**GET** از اول Idempotent هست.  
اما Codexiuo هدفش اینه که **POST** هم بشه Idempotent.

---

## 🦾 راه‌حل: Idempotency Key  
کلاینت یه کلید یونیک می‌فرسته:

```http
POST /checkout
Idempotency-Key: 8f23-ds2a-1bb9-xyz
```

سرور چک می‌کنه:
- آیا قبلاً با این Idempotency-Key درخواست اومده؟
- اگه آره → همون نتیجه قبلی رو بده  
- اگه نه → درخواست رو انجام بده و نتیجه رو ذخیره کن

---

## 🗂 مثال ساده با Express/TS

```ts
import express from "express";
const app = express();

const store: Record<string, any> = {};

app.post("/pay", async (req, res) => {
  const key = req.headers["idempotency-key"] as string;

  if (!key) return res.status(400).send("Idempotency-Key required");

  if (store[key]) {
    return res.send(store[key]); // نتیجه قبلی را بده
  }

  const result = {
    status: "success",
    transactionId: Math.random().toString(36).slice(2),
  };

  store[key] = result; // ذخیره نتیجه

  res.send(result);
});

app.listen(3000);
```

> توضیح: این نمونه خیلی ساده‌ست، توی واقعی‌ها نتیجه داخل Redis یا DB ذخیره می‌شه.

## 🧪 اشتباهات رایج (Codexiuo-style roasting)

### ❌ ذخیره نکردن نتیجه
«مگه می‌خوای هر بار از اول پول کم کنی؟»

### ❌ یونیک نبودن Idempotency-Key
«اسم کلید رو گذاشتی test123؟ واقعاً؟»

### ❌ اشتراک‌گذاری نکردن State در چند سرور
«یکی می‌گه پرداخت شد، اون یکی می‌گه نشد!»

### ❌ تا زمان نهایی شدن کار، جواب سریع ندادن
«کل سیستم رو منتظر نذار، همه‌مون کار داریم!»

## 🧠 نکته طلایی

Idempotency یعنی:
- کار کاربر امن‌تر  
- API قابل اعتمادتر  
- پرداخت‌ها منطقی‌تر  
- دولوپر کمتر فحش می‌خوره
